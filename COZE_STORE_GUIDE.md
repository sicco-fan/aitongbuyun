### 步骤 2: 检查部署配置

当前 `.coze` 配置已就绪：
```toml
[deploy]
build = ["bash", ".cozeproj/scripts/prod_build.sh"]
run = ["bash", ".cozeproj/scripts/prod_run.sh"]
build_app_dir = "./client"
```

### 步骤 3: 发布操作

1. **登录 Coze 平台**
   - 访问你的 Coze 项目页面

2. **点击「部署」或「发布」**
   - 选择发布到商店

3. **填写应用信息**
   - 上传应用图标
   - 填写应用名称和描述
   - 选择分类和标签
   - 上传截图

4. **提交审核**
   - 确认信息无误后提交
   - 等待审核通过

## 三、环境变量配置

发布到 Coze 商店后，需要确保环境变量正确配置。Coze 会自动注入以下变量：

| 变量 | 说明 | 自动注入 |
|------|------|----------|
| `DATABASE_URL` | 数据库连接 | 需配置 |
| `COZE_SUPABASE_URL` | Supabase 地址 | 需配置 |
| `COZE_SUPABASE_ANON_KEY` | Supabase 密钥 | 需配置 |
| `COZE_BUCKET_ENDPOINT_URL` | 对象存储地址 | 需配置 |
| `COZE_BUCKET_NAME` | 存储桶名称 | 需配置 |
| `COZE_PROJECT_ID` | 项目ID | ✅ 自动 |
| `EXPO_PUBLIC_BACKEND_BASE_URL` | 后端地址 | ✅ 自动 |

### 配置方式
在 Coze 项目设置中添加环境变量。
